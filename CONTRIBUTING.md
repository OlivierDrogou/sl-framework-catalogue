# Contributing

## Adding a framework

1. Fork the repo
2. Create `frameworks/your-framework.md` following the [audit protocol](PROTOCOL.md)
3. Every cell must cite source code or official documentation
4. Include a positioning note: what the framework solves, what it leaves open
5. Submit a PR

## Challenging an assessment

If you maintain a framework and disagree with an assessment:

1. Open an issue with the specific column you're challenging
2. Provide counter-evidence (code, documentation, release notes)
3. If the evidence holds, we update the card

The methodology is transparent. Every claim is verifiable.

## Tone

Neutral and respectful. No framework is "bad" — they optimize for different objectives. The catalogue reveals what each leaves open regarding invariant integrity, not what they do wrong.
